# 00. プロジェクト概要

## プロジェクト名

barns

## 目的

barns は、壁面緑化・インテリアグリーン会社の既存顧客向けアフターサポートアプリである。購入後・施工後の顧客が、手入れ方法を確認し、ケアを忘れず、困ったときに会社へ電話相談しやすくすることで、継続的な満足度と信頼を高める。

このアプリは新規顧客獲得を主目的にしない。MVPでは既存顧客の「管理しやすい」「手入れに迷わない」「相談しやすい」を優先する。

## 想定ユーザー

- 壁面緑化を施工済みの個人・法人顧客
- インテリアグリーン、鉢植え、フェイクグリーン、関連資材を購入済みの顧客
- 定期的な手入れや交換タイミングを忘れやすいユーザー
- 電話で相談したいが、相談内容を整理してから問い合わせたいユーザー

## 提供価値

- 購入・施工したアイテムを一覧管理できる
- アイテムごとの手入れ方法を確認できる
- 水やり、剪定、清掃、点検などをリマインドできる
- ケア履歴を残し、状態変化を把握できる
- 壁面緑化パターンを見て、今後の相談材料にできる
- 問い合わせ前に相談内容をドラフト化できる
- 外部問い合わせは電話相談へ自然に誘導できる

## MVPの考え方

MVPは、購入後サポートの体験を小さく成立させることを目標にする。注文、見積、決済、チャット、予約、スタッフ管理は実装しない。データは local-first とし、会社所有サーバー連携は将来の拡張として設計余地を残す。

## 技術方針

- iOS: Swift / SwiftUI
- Android: Kotlin / Jetpack Compose
- Clean Architecture を採用
- MVPの認証は mock authentication を許容
- mock API server を先に用意する計画を立てるが、本タスクでは実装しない
- 画像は任意項目とし、`null` でもUI・データモデルが破綻しない設計にする

## 成功指標の初期案

- 登録済みアイテムを迷わず確認できる
- ケアタスクを作成・完了できる
- ケアガイド到達までの操作数が少ない
- 電話問い合わせ導線が分かりやすい
- 相談ドラフトが電話前の整理に使える
