# 01. 要件定義

## MVPスコープ

### アカウント・ログイン

- メールアドレス形式のIDでログインできる
- MVPでは mock authentication を利用できる
- ログアウトできる
- 将来の本番認証へ移行できる境界を設ける

### アイテム管理

- 購入・施工済みアイテム一覧を表示する
- アイテム詳細でカテゴリ、設置場所メモ、購入/施工日、注意事項、関連ケアガイドを表示する
- 画像がない場合は、カテゴリ名・色・アイコン・テキスト中心で表現する
- MVPでは手動追加・編集は最小限にし、mock data からの表示を優先する

### ケア

- ケアタスク一覧を表示する
- タスクの完了状態を切り替える
- ケアログを表示する
- 水やり、剪定、清掃、点検、交換目安などのガイドを確認できる
- リマインダー通知の設計を行う。MVP実装ではOSローカル通知を想定する

### パターン閲覧

- 壁面緑化パターンを一覧表示する
- パターン詳細で特徴、向いている空間、メンテナンス目安を表示する
- 画像がない場合はテキストと配色で成立させる

### 相談ドラフト

- 問い合わせ前に相談内容を下書きできる
- 対象アイテム、相談種別、本文、緊急度を保存できる
- 送信機能は実装しない
- 外部問い合わせは電話相談へ誘導する

### 会社情報・電話導線

- 会社案内、営業時間メモ、問い合わせ方針を表示する
- 電話番号は本リポジトリに実データを入れない
- アプリ実装時は環境設定または会社提供データから差し込む

## MVP対象外

- アプリ内注文
- 見積依頼の送信
- 決済
- チャット
- 予約・スケジューリング
- スタッフ管理画面
- 本番APIサーバー
- 実顧客データ登録
- 画像アセットの同梱

## 将来機能

- 会社所有サーバーとの同期
- 本番認証
- 顧客ごとの契約・施工情報連携
- メンテナンス提案の自動生成
- 画像アップロード
- スタッフ向け対応履歴管理
- プッシュ通知基盤
- 保証期間・交換時期通知

## 非機能要件

- オフラインでも主要情報を閲覧できる
- 個人情報は最小限にする
- 画像未設定でもUIが崩れない
- iOS/Androidで主要画面構造を揃える
- Clean Architecture によりUI、Domain、Dataを分離する
- 将来のAPI差し替えを容易にする
