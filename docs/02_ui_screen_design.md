# 02. UI画面設計

## デザイン方向性

barns のUIは、清潔感、落ち着き、上質さ、自然感を重視する。購入後サポートのアプリなので、過度に販促的な見せ方ではなく、日常的に安心して使える道具として設計する。

## ビジュアルトーン

- ベース: ホワイト、アイボリー
- メイン: ディープグリーン
- アクセント: ウッドトーン、淡いセージグリーン
- 余白: 広めで落ち着いた密度
- 角丸: 控えめ
- 影: 最小限
- 写真: 任意。画像がない場合は余白、色面、テキスト、カテゴリラベルで成立させる

## 主要画面

### 1. ログイン

- メールアドレス入力
- パスワード入力または mock login ボタン
- MVPでは認証が仮であることを開発環境内で分かるようにする

### 2. ホーム

- 次に必要なケア
- 登録アイテムの概要
- 重要なお知らせ
- 電話相談への固定導線

### 3. マイグリーン一覧

- 購入・施工済みアイテムカード
- カテゴリ、設置場所、次回ケア予定を表示
- `imageUrl` が `null` の場合はカテゴリ色とアイコン表現に切り替える

### 4. アイテム詳細

- アイテム基本情報
- 関連ケアガイド
- ケアタスク
- ケアログ
- 相談ドラフト作成ボタン

### 5. ケアタスク

- 今日/今週/今後の区分
- 完了操作
- タスク種別
- 関連アイテムへのリンク

### 6. ケアガイド

- カテゴリ別ガイド一覧
- 手順、頻度、注意点
- テキスト中心で成立する構成

### 7. パターン閲覧

- 壁面緑化パターン一覧
- 空間タイプ、雰囲気、メンテナンス目安
- 相談時の参考情報として利用

### 8. 相談ドラフト

- 対象アイテム
- 相談種別
- 状況メモ
- 緊急度
- 電話前に確認する内容
- 送信は行わず、電話相談へ誘導する

### 9. 会社情報・問い合わせ

- 問い合わせ方針
- 電話相談ボタン
- 営業時間メモ
- 実住所・実電話番号はこのリポジトリに入れない

## ナビゲーション初期案

- ホーム
- マイグリーン
- ケア
- パターン
- 相談

## アクセシビリティ

- 文字サイズ変更に耐える
- 色だけで状態を伝えない
- タップ領域を十分に確保する
- 画像未設定時も内容理解に支障がない
