# 05. セキュリティ・プライバシー

## 基本方針

barns は既存顧客向けアフターサポートアプリであり、将来的に購入履歴、施工情報、問い合わせ履歴などの個人情報に近いデータを扱う可能性がある。MVP段階から、最小収集、端末内保護、権限最小化、将来のサーバー連携時の安全性を前提に設計する。

## データ最小化

- MVPでは実顧客データを入れない
- mock data は架空データのみ
- 実住所、実電話番号、実顧客名をリポジトリに含めない
- 相談ドラフトには機微情報を書きすぎないようUIで注意喚起する

## ローカル保存

- 端末内DBを利用する場合、OS標準の保護機構を活用する
- 認証トークンはKeychain/EncryptedSharedPreferences等に保存する
- 不要なログ出力を避ける
- バックアップ対象にするデータとしないデータを検討する

## 通信

- 本番APIはHTTPS必須
- 証明書エラーを無視しない
- mock server は開発用途に限定する
- 本番トークンをmock環境に混在させない

## 権限ポリシー

MVPで検討する権限:

- 通知: ケアリマインダーに使用。任意許可
- 電話発信: OSの電話画面へ遷移する場合のみ。自動発信はしない

MVPで原則要求しない権限:

- 位置情報
- 連絡先
- カメラ
- 写真ライブラリ
- マイク
- Bluetooth

画像アップロードなどの将来機能を実装する場合は、その時点で権限目的と代替手段を明示する。

## 認証・認可

- MVPは mock authentication
- 本番ではユーザーごとのデータ分離が必須
- サーバー側で認可を行い、クライアント側判定だけに依存しない
- ログアウト時にローカルセッションを削除する

## プライバシー表示

- 収集する情報
- 利用目的
- 端末内保存の扱い
- 会社への問い合わせ時の扱い
- データ削除方法

これらはリリース前に法務・運用方針と合わせて確定する。

## セキュリティ確認項目

- 秘密情報がリポジトリに含まれていない
- mock data に実個人情報がない
- ログにメール、相談本文、トークンを出さない
- APIエラー時に内部情報を出さない
- 画像URLの外部読み込みポリシーを定義する
