# 08. 法務・リリース・運用

## 法務確認

リリース前に以下を確定する。

- 利用規約
- プライバシーポリシー
- アプリ内問い合わせ方針
- データ削除依頼の受付方法
- 施工写真・商品画像の利用許諾
- ストア掲載情報

## 実データ取り扱い

- リポジトリに実顧客データを含めない
- 実住所、実電話番号、実顧客名を含めない
- 開発用データは架空データのみ
- 本番データは会社管理の安全な環境で扱う

## ストアリリース準備

### iOS

- Apple Developer Program
- Bundle ID
- App Privacy
- TestFlight
- App Store Connect の説明文、スクリーンショット、問い合わせ先

### Android

- Google Play Console
- Application ID
- Data safety
- Internal testing
- ストア説明文、スクリーンショット、問い合わせ先

## 運用方針

- ケアガイドの更新責任者を決める
- お知らせの更新頻度を決める
- 電話問い合わせ先の変更手順を決める
- 障害時のユーザー案内方法を決める
- アプリバージョンごとのサポート期間を決める

## リリース判定チェック

- MVP対象外機能が混入していない
- mock authentication が本番扱いになっていない
- 本番電話番号や住所の扱いが会社承認済み
- プライバシーポリシーが公開済み
- 通知許可の説明が適切
- 画像の利用権利が確認済み
- 画像なし状態で主要画面が成立する

## 初期運用リスク

- 顧客がアプリから注文や見積ができると誤解する
- 電話相談の受付時間外に不満が出る
- ケアガイドが実態と違う場合に信頼を損なう
- 端末内データと将来サーバーデータの同期方針が未確定

これらはアプリ内文言、会社確認、運用設計で早期に潰す。
